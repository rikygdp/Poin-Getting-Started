How to Connect our Backend API?

Request Access Token

Used to get Access Token

URL
{{base_url}}/auth/login
METHOD
POST
HEADERS
Key Value Description
Authorization Basic [Base64Credential] - Basic Authentication = Clients send it’s Base64 encoded credentials with each request, using HTTP [Authorization] header
- Format (Before Encoded) = ClientId:ClientSecret
*For client id and secret registration please contact GPoin Team.
REQUEST BODY
Names Value
grant_type client_credentials
RESPONSES
HTTP Status Description JSON Response
200 Success {
  "access_token": "1234567A-1234-1234-123s-1234567890we",
  "token_type": "bearer",
  "expires_in": 25234683,
  "scope": "read write client",
  "auth_name": "[CLIENT NAME]"
}
401 Unauthorized {
  "timestamp": 1532934913955,
  "status": 401,
  "error": "Unauthorized",
  "message": "Bad credentials",
  "path": "/auth/login"
}

How to generate PSID ?

Generate PSID by Social Media ID

Used to generate PSID for consumer who register through social media on your system, so that can use Poin System

URL
{{base_url}}/api/v1/consumer/add_by_socialmedia
METHOD
POST
HEADERS
Key Value Description
Authorization Bearer [ACCESS_TOKEN]
REQUEST BODY
Names Type
name String The name be used to consumer.
social_media_type String The type of social media used by consumers that registered in your system and it is required.
example ‘facebook’.
social_media_id String This is the id from social media owned by the consumer that registered in your system and it is required.
email String Consumer's Email from Social Media
email_verification_status Boolean Verification Status of Consumer

Generate PSID by Email

Used to generate PSID for consumer who register through email

URL
{{base_url}}/api/v1/consumer/add_by_email
METHOD
POST
HEADERS
Key Value Description
Authorization Bearer [ACCESS_TOKEN]
REQUEST BODY
Names Type
name String The name be used to consumer.
Email String Consumer's Email
How to update User Profile ?

Used to make changes of the consumer profile if consumer wants to change the name and email or one of them. Changes can not be made by the consumer with unverified email address and to do it, you must be login.

URL
{{base_url}}/api/v1/consumer/profile
METHOD
POST
HEADERS
Key Value Description
Authorization Bearer [ACCESS_TOKEN]
REQUEST BODY
Names Type
name String Set it be your name. This is an optional data,
if you don't want to update the user name, you don't have to include this data in the API.
email String Set it be your email. This is an optional data.
The updated email should verified first in your application.
